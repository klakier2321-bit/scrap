# Feature Store

## Cel

Feature store ma przechowywać wersjonowane, futures-aware definicje cech.

## V1

W V1 nie budujemy jeszcze ciężkiej infrastruktury. Budujemy kontrakt:

- nazwa cechy
- wersja
- źródło danych
- częstotliwość
- zależności
- jednostka
- interpretacja
- znane ograniczenia

## Typy cech

- momentum
- mean reversion
- volatility state
- breakout context
- volume anomaly
- funding imbalance
- relative strength
- regime-sensitive features
